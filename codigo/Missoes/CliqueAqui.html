Esta funcionalidade depende do site funcional. Ou seja, ela só funciona integralmente com base no que foi planejado quando o aplicativo está completamente desenvolvido e com todas as funcionalidades implementadas. Portanto, para fim de testes e avaliação na Sprint 3, disponibilizei essa funcionalidade separadamento com controladores que simulam o seu funcionamento.

Essa funcionalidade surgiu da necessidade de manter o usuário engajado no aplicativo, fornecendo um certo estímulo para que ele conclua mais fases e questões. Para que esse objetivo fosse atendido, foi pensado na implementação de missões diárias e semanais, que geralmente requerem que o usuário complete um X número de fases ou que ele consiga uma X quantidade de Moédas. Desse modo, o usuário se sente motivado a realizar as fases do site.

Dados Json
A fim de obter um banco de questões e gerar questões aleatórias todos os dias, usei um JSON para guardar as questões
Geração de Questões Aleatórias

dailyQuests[Math.floor(Math.random() * dailyQuests.length)]

dailyQuests é todo objeto de missões diárias, que tem 7 elementos/missões.
O código acima gera um número aleatório entre 0 e o tamanho do objeto dailyQuests e usa esse número como índice para acessa o elemento no índice do número aleatório.

Carregamento dinâmico de Missões
O JavaScript carrega missões de um arquivo db.json usando fetch e exibe uma missão diária e uma semanal aleatoriamente. Isso é feito na função loadRandomQuests, que seleciona uma missão aleatória e chama createQuestCard para criar o cartão de missão.
Atualização de Progresso
Funções como checkCompletion verificam se a missão foi completada ao atualizar a barra de progresso. Se a missão é concluída, o cartão é atualizado para refletir isso visualmente.
Uso do Bootstrap
A aplicação utiliza Bootstrap para estilizar os componentes, aproveitando classes como btnbtn e progressprogress para criar uma interface responsiva e visualmente agradável.
